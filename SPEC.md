## 1. 問題陳述

### 1.1 目標使用者

| 族群 | 規模 | 痛點 |
|---|---|---|
| 北漂/南漂青年 | ~50 萬/年 | 跨平台重複搜尋、資訊分散、不知哪個平台真實可靠 |
| 換屋家庭 | ~15 萬/年 | 需要比較坪數/租金/學區/通勤，傳統網站 filter 不夠直覺 |
| 外籍租屋者 | ~3 萬/年 | 語言門檻高、房東/室友文化不了解 |
| 海外工作回台 | ~5 萬/年 | 短期需求、不熟悉平台機制、易被騙 |

### 1.2 為什麼不做替代方案

- **全人工代尋**：每筆收費 1,500-3,000 NT$，單次性、不可重複使用
- **單一平台 App**：591/樂屋網/信義/永慶 4 個平台房源不互通，使用者必須裝 4 個 App 並重複登入
- **我們的解法**：單一介面聚合 4 個平台，並提供進階搜尋 + 房東評價 + 看屋助手 checklist 三項增值

---

## 2. 解決方案

### 2.1 核心價值主張

> 「找房子，不用 4 個網站跳來跳去 — 一個搜尋框，全平台房源一次看完。」

### 2.2 使用者流程

1. 進入首頁，於單一搜尋框輸入「區域 + 預算 + 房型」
2. 系統同時呼叫 4 個平台 API（CORS proxy）回傳房源列表
3. 使用者以表格/地圖模式瀏覽，套用 filter（坪數、樓層、屋齡、押金）
4. 點選任一房源查看完整細節 + 房東評價 + 相似物件比較
5. 一鍵加入「我的最愛」並啟用看屋助手 25 項 checklist

## 3. 功能清單

### 3.1 MVP（必做）

- [ ] 4 平台聚合搜尋（591/樂屋網/信義/永慶）
- [ ] 統一 filter 條件（區域/租金/坪數/房型/樓層/屋齡）
- [ ] 表格檢視 + 地圖檢視雙模式
- [ ] 房源最愛收藏（localStorage）
- [ ] 看屋助手 25 項 checklist（含列印版）

### 3.2 v2（加值）

- [ ] 房東評價系統（使用者上傳評論 + 黑名單標記）
- [ ] 通勤時間計算（串接 Google Maps API）
- [ ] 相似物件 AI 推薦
- [ ] LINE 推播新房源通知

### 3.3 明確不做

- 線上簽約（金流/法律合規複雜，超出 v1.0 範圍）
- 房東刊登後台（目前僅聚合公開資料）
- 短租 Airbnb 房源（非目標族群）
- 房屋貸款試算（與銀行端 API 整合需授權）

## 4. 技術棧

| 層 | 選擇 | 理由 |
|---|---|---|
| 前端 | React 18 + Vite + TypeScript | 純 SPA、開發速度快、bundle size 小 |
| 後端 | Vercel Function (CORS proxy) | 解決 4 平台 CORS 限制、免費額度足夠 |
| 資料持久化 | localStorage | 無個資疑慮、使用者自有資料 |
| 地圖 | Leaflet + OpenStreetMap | 免費、開放原始碼、不需 Google API key |
| 部署 | Vercel | 與既有 41 個專案一致、Hobby 計畫免費 |

## 6. 風險與決策

| 風險 | 等級 | 緩解 |
|---|---|---|
| 4 平台 CORS 政策變動導致 proxy 失效 | 🔴 高 | Vercel Function 為單一可控層；建立 fallback 直接呼叫（若 proxy 失效時提示使用者） |
| 平台 API 變更（schema breaking） | 🟠 中 | 抽象化各平台 adapter 層；單元測試每個 adapter |
| 房源資料著作權爭議 | 🟠 中 | 僅快取摘要 + 連結回原平台，不存完整內容；於條款頁明確聲明 |
| 使用者個資疑慮（最愛清單） | 🟡 低 | 全 localStorage、無後端儲存；隱私頁說明 |
